# Top-down Maze Game — Design Spec

**Date:** 2026-05-23
**Status:** Approved for implementation planning

## Цель

Браузерная top-down игра: игрок ходит по процедурно генерируемому лабиринту, ищет выход. На пути — монстры разных типов, цветные двери с ключами, сундуки со случайным содержимым (power-up или debuff). Игровая зона ограничена «кругом видимости» вокруг игрока (fog of war), уже исследованные клетки запоминаются приглушённо. Управление — клавиатура+мышь и Xbox-геймпад через Gamepad API.

## Стек

- **Phaser 3** — движок, подключается через CDN-скрипт в `index.html`.
- **Vanilla ES-модули** в браузере, без сборщика на старте. Если станет тесно — Vite.
- **Pixel-art спрайты**, размер тайла 32×32. На MVP — сгенерированы программно через `Graphics.generateTexture`, позже можно заменить на ассеты из Kenney pack.
- **Локальный dev-сервер** для запуска: `python3 -m http.server` или аналог.

Никаких runtime-зависимостей кроме Phaser. Никакого build-шага на MVP.

## Архитектура

### Файловая структура

```

    GameOverScene.js
    VictoryScene.js
  world/

    TileMap.js         # tile-grid → физические тела стен

  entities/
```

### Модель мира

Двуслойная:

- **Логический слой** — 2D-массив `tiles[y][x]` со значениями `WALL`, `FLOOR`, `DOOR_R/G/B`, `ENTRANCE`, `EXIT`. Размер ~31×21 (нечётные — удобно для алгоритма). При тайле 32 px игровое поле ≈ 992×672, помещается в типичный 1024×768 viewport.
- **Физический слой** — каждая `WALL` превращается в статический `Rectangle` в группе Phaser arcade-physics. Соседние стены не сливаются (оверхед ничтожен на 31×21). Движение игрока, монстров и пуль — пиксельное, коллизии AABB.

Pathfinding монстров работает по тому же логическому слою через BFS: непроходимо = `WALL` или непохожая дверь.

### Состояние игры

Простой синглтон `GameState` в памяти `GameScene`:
- HP, броня, stamina (числа)
- патроны
- инвентарь ключей (Set цветов)
- активные эффекты (массив с тайм-аутами)
- ссылка на текущий лабиринт и игрока

Без внешних state-менеджеров.

## Управление и ввод

`Input.js` каждый кадр опрашивает все источники и возвращает унифицированный объект:

```
{
  move: {x, y},        // -1..1, нормализован
  aim:  {x, y} | null, // направление прицела
  sprint: bool,        // удержание
  dash: bool,          // edge (одиночное)
  shoot: bool,         // удержание = авто-fire
  interact: bool,      // edge
  pause: bool,         // edge
}
```

Игровая логика не знает, откуда пришёл сигнал.

### Клавиатура + мышь

- `WASD` или стрелки — движение
- Курсор мыши — направление прицела
- ЛКМ — стрельба
- `Shift` — спринт (удержание)
- `Space` — dash
- `E` — взаимодействие
- `Esc` — пауза

### Xbox-геймпад (Gamepad API, standard mapping)

- Левый стик — движение (deadzone 0.15)
- Правый стик — прицел и стрельба (deadzone 0.20; авто-fire при отклонении)
- `LT/L2` — спринт
- `A` — dash
- `X` — взаимодействие
- `Start` — пауза

macOS-нюанс: Xbox-контроллер через Bluetooth определяется браузером как standard mapping, драйверов не нужно. Safari иногда требует первое нажатие кнопки для активации — поллинг `Input.js` это переживает.

### Определение активного устройства

Если за последние 500 мс был сигнал с геймпада — активен геймпад (показываем подсказки кнопок геймпада в UI). Иначе — клавиатура. На сам ввод не влияет (читаются оба, объединяются).

## Генерация лабиринта

**Алгоритм:** recursive backtracker — даёт perfect maze (ровно один путь между двумя клетками), без петель. Это критично: ключ-перед-дверью никогда не «обходится».

**Шаги:**
1. Вход — случайный leaf-узел у края.
2. Выход — leaf с максимальной BFS-дистанцией от входа (дальний угол).
3. Двери: ищем bridge-рёбра на пути вход→выход; ставим 1–3 цветные двери на наиболее узких местах.
4. Ключи: для каждой двери — ключ в случайной клетке поддерева, доступного до этой двери. Гарантирует достижимость.
5. Pickup'ы (аптечки, патроны) — равномерно по тупикам (dead-end клеткам).
6. Сундуки — несколько штук в тупиках, отдельно от обычных pickup'ов.
7. Монстры — 5–8 штук в случайных клетках на расстоянии ≥ N тайлов от входа.

## Fog of War

Поверх сцены — `Graphics`-маска чёрного слоя с круглой «дыркой» радиусом ~5 тайлов вокруг игрока, мягкий градиент по краям через `RenderTexture`. Каждый кадр перерисовывается (дёшево).

Запомненные клетки (где игрок бывал) — отрисованы приглушённым серым, текущая видимая зона — полным цветом. Классический «explored vs visible».

## Бой и сущности

### Игрок

- HP: 3. 1 урон за касание монстра или попадание дальнобойной атаки.
- **Броня**: 0 по умолчанию, до 2 единиц. Поглощает урон вместо HP. Регенерирует +1 через 8 с вне боя (без полученного урона). Подбирается из сундука.
- **Stamina**: 0–100%. Спринт тратит ~40%/с, отдых восстанавливает ~25%/с. Без stamina спринт не активируется.
- **Dash**: 200 px в направлении движения за ~120 мс, i-frames на всё время рывка, проходит сквозь монстров (но не сквозь стены). Cooldown 1.5 с. Это и есть «перепрыгнуть монстра».
- **Боеприпасы**: пули — расходник, стартовое количество балансируется. Магазины по 6 в тупиках и сундуках.
- **Ключи**: в инвентаре, иконки в UI. Касание двери своего цвета — дверь открывается, тайл становится `FLOOR`.

### Стрельба

Twin-stick: пока есть направление прицела и патроны — раз в ~300 мс летит пуля. Скорость ~500 px/с, время жизни 0.8 с. Об стену исчезает, о монстра — 1 урон и исчезает.

### Монстры (3 типа)

| Тип | Скорость | HP | Поведение |
|---|---|---|---|
| **Бродяга** | 90% игрока | 1 | Случайные блуждания. Замечает игрока в радиусе 4 тайла → chase BFS. Теряет из виду через 3 с — обратно к блужданиям. Самый частый. |
| **Преследователь** | 70% игрока | 2 | Всегда chase к игроку через BFS, независимо от видимости. «Неотвратимый», нужен для напряжения. |
| **Сторож** | 0 (стационарный) | 2 | Охраняет зону 3×3 вокруг точки спавна. Игрок вошёл — атакует, вышел — возвращается. Блокирует ключевые проходы. |

BFS перестраивается раз в ~250 мс, не каждый кадр.

Касание монстра = 1 урон + knockback на 0.4 с + i-frames на этот период (нет «застрял-умер»).

### Сундуки

Закрытая иконка на полу. Взаимодействие (`E`/`X`) открывает — выпадает случайное содержимое:

- **70% — power-up:**
  - Аптечка (+1 HP, до максимума)
  - Магазин патронов (+6)
  - Броня (+2)
  - Приманка (1 шт. в отдельный слот)
  - Очки-компас (20 с)
- **30% — debuff:**
  - Отравление: −1 HP за 4 с (всего 4 HP за 16 с, если не лечиться)
  - Замедление: −30% скорости на 8 с
  - Слепота: радиус видимости вдвое меньше на 6 с

Игрок не видит содержимое заранее — слепой выбор risk/reward.

### Power-ups с временной активацией

- **Приманка**: бросок по нажатию (отдельный слот), приземляется на 6 тайлов вперёд. На 5 с все монстры в радиусе 8 тайлов переключают цель с игрока на приманку. По истечении возвращаются к обычному поведению.
- **Очки-компас**: пассивные на 20 с. Мигающая стрелка на краю круга видимости указывает направление к выходу.

## UI (UIScene поверх GameScene)

- HP — сердечки в левом верхнем углу
- Stamina-бар под HP, узкая полоска
- Броня — щитки рядом с HP, если есть
- Боеприпасы — иконка пули + число
- Ключи — маленькие цветные иконки
- Активные эффекты — иконки с круговым таймером
- Индикатор активного устройства ввода с подсказками текущих доступных действий
- Компас (когда активны очки) — стрелка на краю круга видимости

## Состояния (сцены)

1. **BootScene** — preload/generate ассетов
2. **MenuScene** — заголовок, «Start», «Controls». Поддержка обоих устройств ввода
3. **GameScene + UIScene** — основной геймплей
4. **GameOverScene** — «Вы погибли», статистика (время, монстры убиты, % исследовано), кнопка «Заново» → новая генерация
5. **VictoryScene** — «Вышел!», статистика, кнопка «Ещё раз»

## Этапы реализации (от простого к сложному)

Каждый этап даёт играбельный билд: то, что собрано, работает.

| # | Этап | Что появляется | Можно проверить |
|---|---|---|---|
| 0 | **Скелет** | `index.html` + Phaser CDN + пустая GameScene с фоном | Сцена рисуется |
| 1 | **Фиксированный лабиринт + игрок** | Захардкоженная сетка 31×21, игрок с WASD-движением, AABB о стены | Бегаем по стенам, не проходим сквозь |
| 2 | **Вход и выход** | Тайлы ENTRANCE/EXIT, касание EXIT → VictoryScene | Доходим до выхода — побеждаем |
| 3 | **Процедурная генерация** | `MazeGenerator` с recursive backtracker, вход/выход по BFS-дистанции | Каждый рестарт — новый лабиринт |
| 4 | **Fog of war** | Маска вокруг игрока + memory of explored | Видим только круг, остальное темнота |
| 5 | **Геймпад** | `Input.js` объединяет клавиатуру/мышь и Gamepad API | Xbox-геймпад управляет так же |
| 6 | **Первый монстр** | Один тип «преследователь», BFS, касание = смерть → GameOver | Бегаем от монстра |
| 7 | **Боевая система: HP, i-frames, knockback** | HP=3, аптечки в тупиках | Не умираем с одного касания |
| 8 | **Стрельба twin-stick** | Пули по мыши/правому стику, патроны | Стреляем в монстра — убиваем |
| 9 | **Спринт и dash** | Stamina-бар, рывок с неуязвимостью | Перепрыгиваем монстров |
| 10 | **Три типа монстров** | Бродяга, преследователь, сторож с разным поведением | Видим разнообразие |
| 11 | **Двери и ключи** | Цветные двери + ключи + размещение в bridge-рёбрах | Нужно искать ключ |
| 12 | **Сундуки с power-up/debuff** | Слепой выбор, броня, приманка, очки, отравление, замедление, слепота | Risk/reward механика |
| 13 | **Полировка** | Звуки (опционально), частицы, экраны меню/Game Over/Victory, статистика | Игра ощущается законченной |

Этапы 0–4 — самостоятельная мини-игра «найди выход в темноте». 5–9 — добавляют action. 10–13 — глубина и разнообразие.

## Что осознанно исключено

- **Мультиплеер** — нет.
- **Сохранения** — нет; каждая игра — отдельная сессия.
- **Звук** — опционально на этапе полировки, не критичен для MVP.
- **Мобильное управление (тач)** — не в скоупе MVP, только клавиатура+геймпад.
- **Несколько уровней с прогрессией** — один процедурно сгенерированный лабиринт за запуск. Реиграбельность за счёт генерации.
- **Сложный balance/прокачка** — параметры из `config/constants.js`, балансируется итеративно во время полировки.

## Открытые вопросы для этапа балансирования

Решаются во время этапов 6–12, не блокируют дизайн:
- Стартовое количество патронов.
- Точный размер круга видимости.
- Точные коэффициенты stamina и cooldown'ов.
- Распределение типов монстров (сколько каких).
- Количество сундуков на лабиринт.
